# Commety Core

## Mission

The Commety Core is responsible for building and maintaining the most coherent representation of reality through independent observations, verifiable evidence and explainable decisions.

The Core does not manage UI, databases or external services.

Its only responsibility is to model the domain and make coherent, explainable decisions.

---

# Core Principles

## Reality comes first

Reality exists independently from Commety.

The platform does not create reality.

It observes it.

---

## Incident-Centric Architecture

The Incident is the central entity of the domain.

Everything else exists to understand, enrich and evolve an Incident.

---

## Observations are independent

Every source contributes observations.

Sources may include:

- Users
- Artificial Intelligence
- Public Authorities
- Sensors
- Webcams
- Radar
- Future integrations

No single observation represents reality.

---

## Evidence builds knowledge

Observations are analysed by the Core.

From those analyses the Core produces Evidence.

Evidence increases or decreases the coherence of the current representation.

---

## Trust is a consequence

Trust is never assumed.

It emerges from the coherence of the available evidence.

---

## Decisions must be explainable

Every decision taken by the Core must always answer one question:

"Why?"

The reasoning behind every decision must always be reconstructable.

---

# Domain First

Technology never defines the domain.

The domain defines how technology is used.

The Core must remain independent from:

- Next.js
- React
- Firebase
- Firestore
- APIs
- External services

Technologies may change.

The Domain must remain stable.

---

# Golden Rule

Before writing code, ask:

> Which domain concept does this represent?

If the answer is unclear, stop.

Improve the domain before writing the implementation.
